# 工具入口默认隐藏并通过悬停显现

此决策已被 `docs/adr/0004-codex-like-manual-sidebar-layout.md` 取代。新的方向采用用户手动控制的功能侧栏，不做鼠标悬停自动显隐。

阅读器的工具入口默认隐藏，只有当鼠标移动到对应触发区域时才展开。这个选择服务于摸鱼小窗阅读场景：平时窗口更像一块干净正文，减少被旁人注意到的界面 chrome；代价是可发现性降低，因此触发区域、展开动画和必要状态提示必须足够清楚。

**Considered Options**

- 始终显示一条极轻顶部工具栏
- 默认隐藏工具入口，鼠标悬停到对应区域时展开

**Consequences**

- 原型必须验证无工具状态下正文是否仍能让用户知道当前阅读上下文。
- 目录、设置和老板键入口需要有稳定的悬停触发区域。
- 小窗模式下不能依赖常驻按钮完成核心操作。
- 后续实现需要处理鼠标进入、离开、延迟收起和键盘可达性。
