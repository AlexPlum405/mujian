# 接入网络书源：发现 + 下载到本地 + 在线流式阅读

木简从"纯本地 TXT 阅读器"扩展为"本地 TXT + 网络书源"双模式阅读器。支持导入社区通用的 JSON 书源（Legado 兼容格式），实现书源发现、整本下载到本地、以及在线流式阅读三种使用路径。

**Considered Options**

- 只做"书源发现 + 下载到本地"，不做流式阅读
- 完整方案：发现 + 下载 + 在线流式阅读
- 不接入网络书源，保持纯本地

**Decision**

采用完整方案。理由：用户明确需要"边读边抓"的在线阅读体验，且下载到本地可作为离线兜底。纯本地方案无法满足找书需求。

**Consequences**

- **定位改写**：`CONTEXT.md` 的"小说阅读器"词条从"本地 TXT"扩展为"本地 TXT + 网络书源"。`_Avoid_: 阅读平台` 重新定义为"不做书城、社区、推荐流，但做书源接入"。
- **数据模型重构**：`NovelChapter.body` 从现成的 `String` 改为可异步获取的形式（懒加载 + 缓存）。原有"打开本地 TXT 即读"必须保留为兼容路径。
- **新增基础设施**：
  - HTTP 客户端（`URLSession`，默认走系统代理，无需自建代理配置）；
  - HTML 解析（引入第三方依赖 `SwiftSoup`，这是木简首次引入第三方依赖）；
  - 书源规则引擎（解析并执行 Legado 兼容的 JSON 书源规则，含 JSONPath/CSS 选择器/正则）；
  - 章节缓存层（在线正文落盘，支持离线阅读）。
- **持久化升级**：`UserDefaults` 升级为数据库（SwiftData/SQLite），因为在线书需落库管理（书源、书 ID、已抓章节、进度）。"书库管理"从可选功能变为必做依赖项。
- **联网与隐私**：需配置 `NSAppTransportSecurity`（部分书源为 http）；老板键隐藏时需支持暂停后台抓取，避免暴露。
- **代理支持**：`URLSession` 默认读取系统代理配置，用户开启代理软件（系统代理模式）即可访问需代理的书源；无需木简自行实现代理。网络失败时需将 `URLError` 翻译为友好提示（如"无法连接，请检查网络或代理"）。
- **工作量量级**：核心模型与持久化层重写，预计代码量翻倍以上。属于多 session 构建，需走 PRD → 拆 issue → 逐个实现的流程。
