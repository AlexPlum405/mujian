# 先做单个可交互 HTML 原型

页面设计阶段先产出一个高保真、可交互的 HTML 原型，而不是先做多套视觉方向或直接进入 SwiftUI 实现。当前布局方向已经由 Codex 式手动功能侧栏收束，继续做三套大分叉会拖慢节奏；单个原型更适合快速验证小窗态、侧栏展开态、侧栏收起态、阅读设置和老板键设置。

**Considered Options**

- 先做 2-3 个视觉方向
- 先做一个可交互 HTML 原型
- 直接开始 SwiftUI 实现

**Consequences**

- 原型需要覆盖真实交互状态，而不只是静态截图。
- SwiftUI 实现以原型结论为准，避免边实现边重新设计。
- 视觉探索收束在同一方向内做微调，不再发散成多套产品形态。
