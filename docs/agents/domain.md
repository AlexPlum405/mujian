# 领域文档

本仓库使用单上下文领域文档布局。

## 探索代码前先读取

工程技能在探索代码、设计功能、排查问题或编写测试前，应优先读取以下文件：

- 根目录的 `CONTEXT.md`，如果它存在
- `docs/adr/` 下与当前工作相关的架构决策记录，如果它们存在

如果这些文件不存在，继续工作即可，不需要特意报错。领域文档可以在项目术语或架构决策逐渐稳定后再补充。

## 推荐结构

```text
/
├── CONTEXT.md
├── docs/
│   ├── adr/
│   └── agents/
└── src/
```

## 使用项目词汇

当输出 issue 标题、重构建议、测试名或设计说明时，优先使用 `CONTEXT.md` 中定义过的领域词汇。不要随意替换同义词。

如果需要使用的新概念还没有写进 `CONTEXT.md`，这通常说明项目词汇还没有定型，可以在后续领域建模时补充。
